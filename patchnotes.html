Hooked Prototype Update - v0.5.74

This build focuses on getting the prototype into a cleaner, more playable state before the next character-model pass. The biggest work is around the skeletal player, grapple visuals, camera control, start-platform readability, and removing systems that were adding friction instead of helping the game feel better.

G

Gameplay and Movement

Grapple and Camera

  • Added manual gameplay zoom on the left side of the screen with mouse-wheel sync.
  • Manual zoom now persists through pause and player death instead of being reset unexpectedly.
  • Improved grapple targeting rules so the player cannot target falling anchors or invalid downward grabs.
  • Added limp chain/rope behavior for failed grapples while falling.
  • Refined the plasma grapple beam with a brighter cyan core, crackle, and lighter pixel-noise treatment.
B

Build Mode and Level Editing

Animation Tools

  • Added an in-game character pose/animation editor for inspecting and adjusting the current paused pose.
  • Added pose frame saving, clear-frame controls, rotate controls, and undo support for animation tweaking.
  • Added limited animator camera orbit so 2.5D poses can be inspected from better angles.
  • Build mode now freezes the ribbon simulation so the attachment point is easier to inspect.
M

Character and Rigging

Skeletal Player

  • Removed the old placeholder character renderer and moved gameplay onto the GLB skeletal player.
  • Added runtime checks for skeleton hierarchy, left/right side semantics, hinge axes, and rigid limb segment lengths.
  • Added authored pose references for idle, launch, swing, release, falling, and flourish states.
  • Adjusted the rope origin to use the left wrist anchor instead of the body center.
  • Improved leg, ankle, and flourish tuck behavior for more readable swing silhouettes.
V

Visual Direction

World and Effects

  • Replaced the original start block with steam stacks, pipes, and procedural smoke.
  • Lowered the start-platform art so the player reads as standing on the ledge instead of clipping through it.
  • Added a darker bottom-screen gradient for a heavier, more ominous composition.
  • Added moon lighting and brighter rim treatment for clouds crossing the moon.
  • Removed pixel-render controls for now so the visual direction feels less gimmicky.
  • Removed the procedural background music while keeping SFX intact.
F

Fixes and Technical Updates

Major Fixes

  • Fixed stale cache-busting references so the live page loads the current v0.5.74 game code.
  • Added performance and character-runtime diagnostics for tracking frame cost, draw calls, and rig health.
  • Changed the end screen to show fastest speed and completed flips for the run.
  • Added malfunction logic for drone and flying-car anchors, including falling behavior and thruster/rotor sparks.
  • Kept the prototype deployable as a lightweight static web build.
