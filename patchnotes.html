Hooked Prototype Update - v0.4.1

Hooked is now a more complete swing-and-stunt testbed. The build adds a finish flow, score/multiplier scaffolding, slow-motion rings, a richer cyber-steampunk backdrop, and a dedicated character rig editor for building the final animated player.

G

Gameplay and Movement

Grapple Feel

  • Improved anchor selection to favor forward momentum and reduce accidental backward grabs.
  • Tuned rope extension, release pop, damping, launch flow, and re-grab behavior.
  • Added a bright cyber rope with a white-hot center, pulsing energy, crackle, and physics sparks.
  • Added a build-mode grapple range circle around the player for easier node placement.

Slow Motion and Flourish

  • Added slow-motion power-up rings that trigger after release instead of wasting time while attached to a grapple.
  • Changed slow motion so its timer only counts down while the player is airborne and not connected.
  • Slowed player movement during slow motion while allowing flourish tricks to play faster relative to the slowed world.
  • Locked flourish input so repeated clicks no longer restart the flip animation mid-flourish.
  • Standardized the current flourish to a tucked backflip.

Scoring and Finish

  • Added score, fail counter, highest multiplier, multiplier timer, and score-rank placeholders.
  • Added the Level Complete panel when landing on the yellow-topped finish building.
  • Added a bottom-center version label showing the current prototype version.
  • Added the celebratory "HEY JEREMY!!!!" scoreboard callout with physics-based fireworks.
B

Build Mode and Level Editing

Object Tools

  • Added build mode controls that only appear while paused.
  • Added add, remove, drag, and place support for anchors, obstacles, bonus rings, slow-motion rings, and the finish line.
  • Added local layout persistence and reset controls for level iteration.
  • Added zoom in, zoom out, and birds-eye view controls for reviewing the whole stage.
  • Adjusted build UI layout so editor controls do not cover core HUD panels as heavily.
M

Character and Rigging

Player Model

  • Recolored the placeholder character closer to the gold-and-black "m" character direction.
  • Moved grapple origin from the body center toward the character's grapple arm.
  • Added two red ribbon trails with chain-style physics behavior and slow-motion response.
  • Improved swing poses so the grapple arm tracks rope angle and the body responds to downswing and upswing positions.

Character Rig Editor

  • Added a standalone character rig editor page.
  • Added SVG part loading with hidden joint markers for alignment.
  • Added layer ordering, multi-select, group/ungroup, drag-select, rotate handles, resize handles, import/export, and undo.
  • Added animation pose saving and a focused looping preview panel.
  • Stored the current baseline pose as m_mid_swing.
V

Visual Direction

World and Background

  • Shifted the scene toward a cyber-steampunk sunset palette with darker purples, oranges, and a larger pale moon.
  • Darkened the background so drone lights and gameplay objects remain readable.
  • Moved cloud mass higher/back into the scene so clouds no longer occupy the active play lane.
  • Added procedural sky, moon, cloud, star, and city texture work without adding heavy raster assets.

Drone Anchors and Feedback

  • Replaced plain anchor points with hovering drone anchors.
  • Moved drone lights above the drone head and added matching circular glow for idle, target, and grapple states.
  • Added drone rotor animation, hover motion, catch impact, and rope-tension response.
  • Added speed lines, motion trails, slow-motion timer icon, and larger yellow multiplier display.
F

Fixes and Technical Updates

Performance and Consistency

  • Added fixed-step gameplay physics so swing damping and momentum are less dependent on display refresh rate.
  • Kept the heavier SVG character disabled in gameplay while the rig is still being refined, preserving smoother swing feel.
  • Removed temporary loader experiments that caused the dark-blue HUD-only screen.
  • Kept the game deployable as a lightweight static prototype.

Mobile and Browser Fixes

  • Fixed mobile selection behavior that showed blue handles and copy/paste controls over the game.
  • Improved touch handling for dragging, tapping, flourish, restart, pause, and build controls.
  • Adjusted HUD and controls for mobile portrait and landscape layouts.
  • Added script cache-busting updates so browsers fetch the newest prototype build.

Deployment

  • Added deployment and watch-deploy scripts for the live Hooked directory on prntscrn.dev.
  • Deployed the prototype and patch notes to the same live Hooked folder for mobile testing.
